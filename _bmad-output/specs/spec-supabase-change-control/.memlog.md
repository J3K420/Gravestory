- (capability) CAP-1: Repository tooling distinguishes ordered migrations from verification and retrieval scripts and rejects duplicate, missing, or malformed migration metadata.
- (capability) CAP-2: Operators can determine the intended migration order, prerequisites, verification, and rollback or forward-fix posture from versioned documentation.
- (capability) CAP-3: One-off database and metrics tasks run from version-controlled code with explicit environment requirements and safety classification.
- (constraint) Never connect to, inspect, or mutate the live Supabase project during this batch.
- (constraint) Preserve applied migration files byte-for-byte unless a separate correctness finding requires a forward migration.
- (non-goal) No automatic production migration runner and no claim that the live schema matches repository history without approved verification.
- (event) Coherence validation passed.
- (event) Preservation validation passed against the program audit and soft-delete/data-safety rules.
- (event) Spec finalized after coherence and preservation validation.
- (decision) Every repository SQL artifact and operational task entrypoint must be classified; uncataloged artifacts fail validation.
- (constraint) All existing numbered migration files are immutable by default because live applied state is not inspected; corrections are forward-only.
- (capability) CAP-4: One-off tools require explicit target selection and confirmation and may not silently default to the production Supabase project.
- (constraint) Depends on deterministic verification and pinned local Supabase tooling.
- (capability) CAP-5: Repository migrations execute against a disposable local Supabase stack for Auth, PostgREST, and RLS semantics when bootstrap prerequisites exist; missing prerequisites remain explicit unresolved evidence.
- (decision) Plain PostgreSQL may support limited syntax checks but is not accepted as equivalent Supabase parity.
- (event) Final BMad Blind Hunter, Edge Case Hunter, and Acceptance Auditor re-review passed after triage.
